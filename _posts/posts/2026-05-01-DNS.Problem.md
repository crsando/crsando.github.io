去年为了打WoW换了新电脑，一开始还是继续用 ***Clash for Windows***，今年春节当时我写了一篇文章说[想试试新东西](./2026-02-28-Try-Something-New.md)，那个时候就换成了 ***Clash Verge Rev***

然后就开始不时出现 ***DNS 解析出错***的问题，看日志是这样子的：

```
dns resolve failed: all DNS requests failed, first error: connect failed: dial udp 8.8.8.8:53: connect: An operation on a socket could not be performed because the system lacked sufficient buffer space or because a queue was full.
```

之前每次出问题就是靠重启解决。我一开始还以为是 ***Clash Verge Rev***的问题，或者是我用的机场的问题，尝试了很久都找不到办法。

今天又重新来排查了一下，在 ***claude-opus-4.6*** 的建议下

1. 一开始想可能是 *fake-ip* 的问题，然后换到 *redir-host* 发现无效，又
2. 然后想会不会是 *dns overwrite* 配置不太对，但是和 claude 推荐的对比看看，尝试改改试试，发现问题并不解决。
3. 关闭 *tun* 模式发现还是有很严重的问题，尝试直接 *nslookup www.baidu.com* 也失败，想到会不会是家里路由器的问题
4. 但是家里路由器要是有问题，早就崩了对吧。。。所以最后在 *claude* 的建议下判断只可能是 windows 的资源确实被某个进程消耗光了。
5. 纯查 ***tcp*** 查不出问题，重新给 *claude* 明确的要求，让他帮我写一段代码查一下 ***udp***，终于。。。

![](https://raw.githubusercontent.com/crsando/picgo/main/20260501233615198.png)

卧槽居然是 ***MSI Center*** （微星的控制中心） 发疯占用了 ***15000+*** 的 UDPEndpoint。

果断开始删除，尝试重启。再观察一阵子，希望问题消失吧！

---

为什么以前没出现呢，我怀疑还是和 *fake-ip* 有一点关系，可能是换了 *Clash Verge Rev* 之后，默认就是用 *fake-ip*，导致微星这边估计有一个查询失败不断重试，就耗尽了所有的 *udp* 资源吧。
